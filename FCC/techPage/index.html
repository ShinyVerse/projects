Javascript Dictionary

Introduction

I wanted to include these because I was late to the game on using some of them and they are mega helpful...in my opinion

The basic gist:

You want to log a collection which can be added to and grow at any time. (While the code is in use that is.)

The Set Up:

You are making an object and either creating a new item or updating an existing one.

var dictionary = {};

Then adding what you want to it:

dictionary.anyItemHere = "some kind of entry.";
OR
dictionary[9] = 1;
OR
dictionary["nine"] = 8;

Updating Dictionary Example:

if (dictionary[9] > 0){
dictionary[9] = dictionary[9] + 1
}

Uses:

Good for iterating through lists and returning a count.
dictionary[number/entry] = dictionary[number/entry] + 1 || 1;

This will allow for creation if not in list yet, or will add to current list. If already there the key's value will be overwritten, but the value will remain intact. Cool, huh?

Recursion

Introduction

Recursion. Where do I start with recursion? Maybe a better question would be where do I end or oh god why, WHY?! Basically you call your function again at the end of the very same function with updated info (params) until you've reached an end, say zero or an end of a list.

Example:

var arr = ["lamb","goat","dog","sheep"];

function animalSays(arr){ var savedArr = arr;
var counter = 0;
if (arr[counter] == "dog"){
__console.log("woof");
} else {
__console.log("no such animal here");
__savedArr.shift(arr[counter]);
__counter++;
__animalSays(savedArr);
__}
}

It's a nifty little technique to add to your skillset, after learning I do pop a few of them in my code here and there. The beauty is once you get the swing of them I can guarantee you'll find places to slot them in.

Tips

A little extra:

A few tips I would have appreciated when starting out.

  • You aren't going to know everything, learn what you can always.
  • Step away for a breather after completing something. Whether that be an entire project, learning a new topic or finishing off a tricky piece of code. A moment to unwind can improve your performance.
  • Algorithms, algorithms, algorithms! You'll be thankful you spent quite a lot of time here.
  • Priming: read articles and watch videos or anything related to what you'll be trying to study prior to actually attempting it. Your brain will be excited to start the task.
  • If you hate a certain topic or skill (for me it's Regex) it is a strong indicator you need to spend more time in this area (I really do and currently am. It's not so fun, but I know it will get better.)
